# 內容寫作 Playbook — 隼訊數位行銷

本文件給未來的內容貢獻者（含 Claude 跟你自己）。目的是讓所有新增的內容**不被 Google 視為 AI slop**、能拿到 E-E-A-T 分數、能真正幫到讀者。

---

## 護欄機制（必須遵守）

### 1. `qualityTier` 三態
每個內容（service / blog / local / pricing / compare）都有 `qualityTier`：

- `placeholder` — 結構在但內容是空殼（不進 sitemap、加 noindex）
- `draft` — 內容寫了但未經審核（不進 sitemap、加 noindex）
- `production` — 完整、有 E-E-A-T 訊號（進 sitemap、可被搜尋）

**預設用 `draft`**。改為 `production` 前須通過下方審核清單。

### 2. `pnpm lint:content` 必通過
本指令會掃描所有 content 檔，偵測 AI slop pattern。HIGH 風險項不通過會 exit code 1，CI 應該擋下 commit。

### 3. CaseStudy.consentToPublish 三態
- `name-only` — 只能寫客戶名、產業、一句話描述
- `metrics-only` — 加上接手日期、量化成果，但不寫客戶內部細節
- `full` — 客戶簽過授權，可寫完整 case study

**沒問客戶 = 預設 `name-only`**。

---

## 內容審核清單（升級為 production 前必須過）

每升一頁為 `production`，下面 8 題必須全部 ✅：

- [ ] 沒有未經來源的統計數字（任何 `N%` `N 倍` `N 個用戶` 都要附 source link 或註明「依我們經驗」）
- [ ] 沒有空泛緊迫感詞（「藍海窗口期」「先發優勢」「等於不存在」「徹底消失」）
- [ ] 沒有競品貶低（提到 Wix / Intercom 等品牌名稱時，不帶評價詞除非有實測證據）
- [ ] 沒有「為什麼 X 必須做 Y」這類 boilerplate 緊迫感標題
- [ ] 沒有「適合做 X 的企業」這類 AI 標誌段落
- [ ] 沒有模板化 CTA（「想知道你的 X？預約免費 X 健診」）
- [ ] 至少有一個「具體」訊號：日期 / 客戶名 / 工具名 / 流程細節 / 限制聲明
- [ ] `pnpm lint:content` 通過

## 訪談題庫（讓你能餵 Claude 寫出有人味的內容）

下面是各種內容類型的訪談題庫。**你只要錄音 / 寫粗稿回答其中 3-5 題**，Claude 就有素材可以幫你整理成有 E-E-A-T 的文字。**不需要全部回答**，質量 > 數量。

### 服務頁訪談（每個服務 5-7 題）

1. 你定 NT$ X / 月這個價格，是因為比某對手便宜，還是因為固定成本（人力、工具）？
2. 過去 12 個月，哪個案最讓你後悔接？發生什麼事？
3. 客戶最常「誤會」這項服務的哪一點？
4. 哪一類客戶你會直接拒絕？為什麼？
5. 你做這個服務時，**對手通常不會做**但你會做的事是什麼？
6. 上一個成交的客戶，他們做決定的關鍵理由是什麼？
7. 過去做這個服務，學到最反直覺的一件事是？

### Case Study 訪談（每個客戶 8-10 題）

1. 接手日期是？合作幾個月？
2. 接手時客戶的痛點是？（具體一兩件事）
3. 你做了哪 3 件最關鍵的事？
4. 過程中遇到什麼意外或卡關？怎麼解決？
5. 客戶簽約前最猶豫的點是什麼？
6. 量化成果：(A) 接手前的數字 (B) 結案 / 目前的數字
7. 客戶最滿意 / 最不滿意的部分？
8. 如果重來一次你會怎麼改？
9. 這個案能否寫成公開 case study？授權等級：name-only / metrics-only / full？
10. 客戶有同意你引用他的一句評論嗎？（一句即可）

### Blog 文章訪談（每篇 4-6 題）

1. 為什麼想寫這篇？你最近遇到哪個情境讓你想寫這個主題？
2. 這個主題，業界主流的看法是什麼？你不同意的點在哪？
3. 你有過一個失敗 / 學到教訓的例子嗎？
4. 如果讀完這篇只能記一件事，你希望是哪件事？
5. 哪些事你刻意**不要寫**？為什麼？
6. 有沒有具體工具 / 資料 / 流程能附上來讓文章可驗證？

## 寫作前後對比範例

### ❌ AI Slop 寫法（會被 lint 抓到）

```
為什麼 2026 年的台灣中小企業必須做 GEO？

AI 搜尋使用率年增 240%，2025 年突破 600 萬月活。
搶占藍海窗口期，享受先發優勢。

5 個關鍵步驟：
1. 部署 llms.txt
2. 加入 Schema
3. ...
```

問題：
- 緊迫感 boilerplate 標題
- 統計沒附來源
- 「藍海窗口期」「先發優勢」是空話
- 「5 個關鍵步驟」湊版面

### ✅ 改寫後（有 E-E-A-T 訊號）

```
GEO 是針對生成式 AI 搜尋的內容優化。

依我們在 2024 年下半年到 2026 年初接案的觀察：
中小企業客戶來找我們做 GEO 的問題通常分兩類 ——
(1) ChatGPT 完全沒提到他們、(2) 同業已經被提到，但他們沒有。

我們的工作流程：
- 先用一組固定問題在 ChatGPT、Claude、Perplexity 測試 baseline
- 部署 llms.txt + 缺漏的 schema
- ⋯
- 兩個月後重測同一組問題，量化變化

不是每個產業都該做 GEO。月廣告預算低於 3 萬的客戶，
我們會建議先把 SEO 跟 GBP 做完再來談 GEO。
```

差別：
- 用「我們的觀察」代替「年增 240%」
- 寫出**工作流程**而非「5 個步驟」
- 寫出「不適合做的客戶類型」 — 真實專家敢於說「不」

## 跨檔協作 — 你 + Claude 的工作流

1. **你**：選一頁要升 production（如 `local/taoyuan-seo`）
2. **你**：回答上面相關訪談 3-5 題（可口述、可隨手寫）
3. **Claude（或你）**：依訪談內容改寫該頁
4. **任一人**：跑 `pnpm lint:content` 確認通過
5. **任一人**：核對審核清單 8 題
6. **你**：把該頁 `qualityTier` 改為 `'production'`
7. **任一人**：commit + 部署

升一頁 production = 一次完整循環。**不要一次升 10 頁**，質量會掉。

## 不該寫的東西（明確禁止）

- 「我們是最 X 的廠商」（最便宜 / 最快 / 最專業）— 自吹自擂無證據
- 「藍海」「先發優勢」「搶占」「黃金窗口期」— FOMO 行銷話術
- 「N 個方法 / N 個秘訣」— 數字湊版面
- 「為什麼必須做 X」— boilerplate 緊迫感
- 「適合做 X 的企業」+ 條列清單 — AI 標誌段落
- 任何「年增 N%」「N 倍」未附 source 的數字
- 「Wix 速度差」「Intercom 太貴」這類無實測證據的競品評價
- 把同樣意思的 CTA 套用到每個頁面（「想知道你的 X？預約免費 X 健診」）

## 想再加新規則？

修改 `scripts/lint-content.ts` 的 `AI_SLOP_PATTERNS` 陣列。每條規則需要：
- `id`：唯一識別字串
- `label`：人看的標籤
- `regex`：偵測用 RegExp
- `severity`：'high' / 'medium' / 'low'
- `reason`：為什麼這是 slop

新增規則後跑 `pnpm lint:content` 看會抓到什麼，調整 regex 直到 false-positive 可接受。
